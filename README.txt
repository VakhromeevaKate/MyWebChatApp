************************************MyWebChat***************************************

Тестовое задание:

Написать web-приложение-чат (если не получится web, то настольное). В качестве механизма 
обмена сообщениями используется СУБД.СУБД - MSSQL. Структура БД на свое усмотрение,
однако обязательно хранение всей истории переписки.

Приложение должно содержать отдельный поток для обращения к серверу и получения новых сообщений.
Интерфейс приложения не должен блокироваться этим потоком - т.е. мы в любой момент должны 
иметь возможность отсылать свои сообщения и оперативно видеть отправленные нам.

Чат можно сделать один общий для всех пользователей.

Использовано:
1. Библиотека Microsoft.AspNet.SignalR;
2. MS SQL Server 2016 Express (параметры подключения только в Web.comfig);
3. MS Visual Studio 2017 Comunity Edition;

Реализовано:
1. Общий чат для всех пользователей в форме web-приложения ASP.NET.
2. Многопоточность на уровне Hub при обращении к серверу.
3. Хранение всей истории переписки в БД и вывод ее на странице History.

Особенности:
Перед развертыванием приложения необходимо создать БД и указать сервер и БД
в web.config в качестве connection string. Таблицы в БД создавать не нужно, 
при первом входе приложение создаст их самостоятельно.
